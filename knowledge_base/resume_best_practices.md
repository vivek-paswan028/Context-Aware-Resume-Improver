# Resume Writing Best Practices

## Action Verbs and Strong Language

### Use Powerful Action Verbs
Start each bullet point with a strong action verb that demonstrates impact:

**Leadership & Management:**
- Led, Directed, Orchestrated, Spearheaded, Championed, Mentored, Supervised

**Technical & Development:**
- Developed, Engineered, Architected, Implemented, Optimized, Automated, Integrated

**Analysis & Strategy:**
- Analyzed, Evaluated, Assessed, Researched, Investigated, Diagnosed, Strategized

**Communication:**
- Communicated, Presented, Negotiated, Collaborated, Facilitated, Mediated, Authored

**Achievement & Results:**
- Achieved, Accomplished, Attained, Exceeded, Surpassed, Delivered, Generated

### Avoid Weak Language
- Replace "Responsible for" with specific action verbs
- Avoid "Helped with" - use "Contributed to" or "Collaborated on"
- Never use "Tasked with" - describe what you actually did

## Quantifiable Achievements

### The XYZ Formula
Structure accomplishments as: "Accomplished [X] as measured by [Y], by doing [Z]"

**Examples:**
- "Increased user engagement by 40% through implementation of real-time notification system"
- "Reduced server costs by 25% by migrating to containerized microservices architecture"
- "Improved code review efficiency by 60% by introducing automated CI/CD pipelines"

### Metrics That Matter
- Percentages (% improvement, % reduction)
- Time savings (hours/week, days reduced)
- Money ($ saved, $ generated, budget managed)
- Scale (users served, requests handled, data volume)
- Quality (error rate reduction, uptime improvement)

## Resume Structure

### Essential Sections (In Order)
1. **Header**: Name, contact info, LinkedIn, GitHub/portfolio
2. **Professional Summary**: 2-3 lines highlighting key achievements and skills
3. **Skills**: Technical skills relevant to target role
4. **Experience**: Reverse chronological order
5. **Education**: Degree, institution, graduation year
6. **Optional**: Projects, Certifications, Publications

### Formatting Guidelines
- Keep to 1-2 pages maximum
- Use consistent formatting throughout
- Font: 10-12pt for body, 14-16pt for headers
- Margins: 0.5-1 inch on all sides
- Save as PDF unless otherwise specified
- Use clear section headers
- Bullet points should be 1-2 lines each

## Content Best Practices

### Tailor Your Resume
- Customize for each job application
- Mirror language from job description
- Highlight relevant experience first
- Remove irrelevant older experience (10+ years)

### Show Progression
- Demonstrate increasing responsibility
- Highlight promotions and role expansions
- Show skill development over time

### Technical Resumes Specifics
- List specific technologies, not just categories
- Include links to GitHub, portfolio, or live projects
- Mention relevant certifications
- Detail technical challenges solved

## Common Mistakes to Avoid

1. **Typos and grammatical errors** - Proofread multiple times
2. **Generic objective statements** - Use targeted professional summary
3. **Listing duties instead of achievements** - Focus on impact
4. **Inconsistent formatting** - Maintain visual consistency
5. **Including irrelevant information** - Keep it job-relevant
6. **Using tables or complex formatting** - ATS may not parse correctly
7. **Including a photo** (in most countries) - Can introduce bias
8. **Listing references** - Provide separately when requested
